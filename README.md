# 15 - 8.5 STREAM ERRORS

## Опис завдання
Мета цієї практичної роботи — реалізувати HTTP-сервер на Node.js, який демонструє навички грамотної обробки помилок у потоках даних (Streams). Основна задача полягає в тому, щоб сервер не припиняв свою роботу (не "падав") у разі виникнення помилок при читанні файлів, наприклад, коли файл відсутній на диску.

## Вимоги до реалізації
*   **Маршрут**: Обробка GET-запитів на `/missing-file`.
*   **Параметри**: Обов'язкова наявність query-параметра `fileName`.
*   **Логіка**:
    *   Спроба відкрити та передати файл клієнту за допомогою `fs.createReadStream`.
    *   Повернення статусу **400**, якщо параметр `fileName` не вказано.
    *   Повернення статусу **500** та зрозумілого повідомлення про помилку, якщо файл не знайдено або сталася помилка стріму.
*   **Стійкість**: Сервер повинен залишатися активним після будь-якої помилки.

## Технічні особливості
Для реалізації використано подію `.on('error')` на об'єкті стріму. Це критично важливо, оскільки в Node.js помилки в асинхронних потоках, які не мають обробника, призводять до завершення процесу (Uncaught Exception).
